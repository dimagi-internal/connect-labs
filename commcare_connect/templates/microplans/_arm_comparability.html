{% comment %} Shared arm-comparison panel. Rendered identically on the study-group page (server-side include) and the single-plan review page (returned by ArmComparabilityView and injected client-side after Generate). Both build the `comparability` context from the same `arm_comparability_psu` engine, so this is the single source of comparison markup. Each arm's row label comes from `comparability.arms[].name` (plan name on the group page; ward name on the plan page). The whole-ward density line renders only when a ward_density is present (group page) and is omitted on the single plan, where it is context-only and 0. Presentation is deliberately OBJECTIVE: per-arm metrics + standardized mean differences as numbers, with no pass/fail verdict, no "match"/"within tolerance" badge, and no per-cell good/ok/bad coloring. The engine still computes `matched` / `incomparable` / the SMD bands (other code, e.g. ArmComparabilityView's JSON, reads `matched`); this panel simply does not render a verdict. The viewer reads the numbers and judges for themselves. {% endcomment %} {% if comparability and comparability.metrics %}
